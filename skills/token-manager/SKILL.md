# Token Manager - 智能Token成本优化技能

## 核心理念

**看到知识，就拥有了知识。这个Skill会自动帮你省钱。**

本技能会在后台自动运行，帮你：
1. 🔄 **自动模型调度** - 简单任务用便宜模型，复杂任务用贵模型
2. 📊 **上下文监控** - 超过阈值自动提醒压缩
3. 🎯 **任务分级** - 自动判断任务复杂度，选择最优模型

---

## 自动运行机制

### 🤖 智能模型调度（核心功能）

**本技能会在每次对话时自动判断任务复杂度：**

| 任务类型 | 复杂度 | 自动选择模型 | 原因 |
|----------|--------|--------------|------|
| 简单问答、信息收集 | 低 | Kimi/国产模型 | 成本低，速度快 |
| 数据整理、格式转换 | 低-中 | Sonnet/GPT-3.5 | 性价比高 |
| 代码编写、复杂推理 | 高 | Opus/Claude | 质量保证 |
| 创意写作、战略决策 | 高 | 顶级模型 | 需要深度思考 |

**自动调度规则**：
```
简单任务（<100字回答）→ 经济模型
中等任务（需要分析）→ 中级模型
复杂任务（代码/推理）→ 顶级模型
```

### 📈 上下文监控

**自动监控对话上下文大小，超过阈值时主动提醒：**

| 上下文大小 | 状态 | 自动行为 |
|------------|------|----------|
| < 50K Token | ✅ 正常 | 无操作 |
| 50K - 100K | 🟡 注意 | 提醒可压缩 |
| > 100K | 🔴 警告 | 强烈建议压缩 |
| > 200K | 🚨 危险 | 立即压缩，否则费用飙升 |

**提醒格式**：
```
⚠️ 上下文提醒
当前上下文：XXX K Token
建议：执行 /compress 压缩记忆
预计节省：50%-80% Token消耗
```

## 三大降本技巧

### 技巧一：善用命令行（零Token消耗）

以下命令**不经过大模型**，直接与系统交互，**零Token消耗**：

| 命令 | 作用 | 适用场景 |
|------|------|----------|
| `/new` | 重启对话，清空上下文 | 对话太长、Token消耗大 |
| `/restart` | 重启整个会话 | 龙虾不理你、卡住 |
| `/stop` | 停止当前任务 | 任务想错了、方向偏了 |
| `/compress` | 压缩记忆文件 | MEMORY.md太大 |

### 技巧二：程序优先，大模型兜底

**原则**：重复任务写成脚本，脚本不行再上大模型。

| 任务类型 | 推荐方式 | Token消耗 |
|----------|----------|-----------|
| 检查邮件 | 脚本轮询 | 0 |
| 看新闻简报 | 脚本抓取 | 0 |
| 数据格式转换 | 脚本处理 | 0 |
| 定时监控 | 脚本+条件触发 | 仅触发时消耗 |
| 复杂推理 | 大模型 | 正常消耗 |

**实施步骤**：
1. 识别重复任务
2. 写成脚本
3. 继承为Skill
4. 下次直接调用

### 技巧三：多模型混用，各司其职

**本技能会自动帮你选择模型，但你也可以手动指定：**

```
"用便宜模型帮我做这个"
"这个任务很重要，用最好的模型"
"启动子agent，用经济模型收集信息"
```

- 🎨 创意写作、重要决策

## 📋 省钱检查清单

每次发现Token消耗异常时，按此清单检查：

- [ ] 记忆文件是否太大？（>100KB → `/compress`）
- [ ] 是否有重复任务可以脚本化？
- [ ] 是否用了过贵的模型做简单事？
- [ ] 对话是否太长？（>50轮 → `/new`）
- [ ] 是否有定时任务在空跑？
